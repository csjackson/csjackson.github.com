Everyone has mornings where they seem to have left their brains behind, and this was one of mine.

The morning run of String Kihon got me 13/14, with my old nemesis, Join, coming back to bite me. There is a static method on the class string that does exactly what I wanted. Running the Join method on the instance (foo.String e.g.) does something entirely different. Confusing! But that sort of thing is what these exercises should be about.  This confusion, though, meant that I did not even attempt the other 2 kihon (side note: what is the plural?).

 

The brain finally engaged somewhere during the day’s work on our adventure program.  We made a Yell command that yanked the text uppercase, and turned it red, since, as Kate pointed out, yells are red.  We reset the foreground color back to default at the end. 

Next came the more vital change to the program: we broke each command out into its own .cs file, and created an enumeration for our commands that we typed “ICommand”, following the convention that enumerations start with Cap I, and named commands.  ICommand, in turn only runs IsValid, and Execute. The real work happened with the following change to Program.cs:

var cmd = commands.FirstOrDefault(c => c.IsValid(input));

                if (cmd == null) cmd = defaultCommand;

                cmd.Execute(input);

 

Let’s take this apart from the inside, out:

1, c is defined, and set with the value of the result of running IsValid on input via a Lambda Construction.

2, cmd is defined and set with the value of the first or default of the commands enumeration

3, If IsValid comes back false, c is set to null, so commands.FirstOrDefault also returns null, so cmd is set to null. We test for this, and if cmd = null, then we set cmd to defaultCommand.

4, Finally, we run the Execute method on cmd, passing in the parameter input.

 

After that, we created 2 more commands, Dance and Wave, which basically follow the same construction:

public void Execute(string input)

        {

            var output = GetAllButFirstWord(input);

            Console.WriteLine(String.Format("You wave at {0}.", output));

        }

 

2 things to note here:

First, the word void means that this method does not return a value to whatever called it.

Second, Writeline(String.Format()) This syntax allows us to plug values, such as the value of output, above, into the writeline statement where the {} are. With just 1 plug in, like we have now, it doesn’t appear to gain us anything over a “blah “ + foo + “ more blah.” construction, but I am confident that having several different things we want plugged in will make this construction seem more attractive.
